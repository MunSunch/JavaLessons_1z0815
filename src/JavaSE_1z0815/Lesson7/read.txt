    Package - это место, где хранятся классы. Выполняет несколько функций:
        1) Access protection. Класс, находящийся в одном пакете, не может обратиться к классу, находящемуся в другом пакете.
    Проблема решается применением модификатора public к последнему и полным именем(Test7_1).
        2) Namespace management. В пакете не существует двух и более классов с одинаковыми именами. В разных пакетах
    такое возможно, потому что полное имя класса будет различаться.
        3) Хранение логически связанных классов в одном месте.
    Package statement должен быть написан всегда. Если он не написан, то класс попадает в default package. Так, он сможет
запускаться, но обратиться к нему с другого класса будет невозможно.

    Access модификаторы
    Используются для сокрытия части данных. Если представить, что код это двигатель машины, то зачем юзеру знать и копаться
в этом двигателе, ведь можно что-то сломать. Существуют следующие модификаторы:
        1) public - элемент доступен везде;
        2) private - доступен внутри класса;
        3) default - доступен внутри пакета, представляет собой пустое обозначение;
        4) protected = default + доступ потомка класса.
    Все четыре типа можно использовать с конструкторами, переменными, методами. С классами можно использовать только
public и default. Стоит учесть, что элементы, находящиеся в методе, локальные элементы, не имеют модификаторов доступа,
так как они существуют только в методе.
