    Scope var
    Границы видимости переменных различаются:
        1) local var видны внутри метода, начиная с момента объявления до конца метода;
        2) parameter var видны внутри метода, начиная с начала тела до конца;
        3) instance var - внутри всего класса;
        4) static var - внутри всего класса.
    При пересечении областей видимости переменных с одинаковым идентификатором для их различения нужно использовать
ключевое слово "this", которое указывает на принадлежность переменной к экземпляру класса(Car.java). This можно написать
со static var, однако это расценивается как бестолковое действие, так как она принадлежит классу.

    Идентификаторы
    Это имена переменных, классов, методов.
    Идентификатор может быть любой длины. Запрещается использовать знаки препинания, логические и арифметические операторы.
    Первый символ - любой(допускаются символы валют), кроме цифры, знака препинания, логического оператора.
    Запрещается использовать зарезервированные keywords.

    У local variable не существует default value по сравнению instance variable.

    Life of Object. Garbage collector
    Обьект удаляется из памяти только в двух случаях: после окончания своего scope и потере связи со своим объектом
(переопределение значений). Удалятором выступает Garbage collector, работа которого непредсказуема. Можно лишь сказать
какие объекты пригодны для удаления, но точно утверждать, что объект будет удален именно в данный момент, к сожалению, нет.
