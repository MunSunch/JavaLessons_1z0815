    Для чего нужен import
    Для обращения к классу, находящемуся в другом пакете и имеющему access mod public, нужно было использовать полное имя класса:
        JavaSE_1z0815.Lesson9.Car c=new JavaSE_1z0815.Lesson9.Car();
    Используем import и пропишем полное имя один раз. Теперь можно при обращении к классу использовать его сокращенное имя.
        import JavaSE_1z0815.Lesson9.Car;
        Car c = new Car();
    При этом импорт из разных пакетов классов с одинаковыми идентификаторами ЗАПРЕЩЁН.
    Для импорта сразу всех классов и пакета нужно использовать * (смотри Test10.java). При импорте звездочкой из двух разных пакетов
для классов с одинаковыми идентификаторами все же придется прописывать полное имя класса как это сделано в том же Test10.java
    import не копирует код, а дает указание JVM о местоположении класса.
    По умолчанию заранее JVM импортируются все классы из пакета java.lang - String, System и др., поэтому, создавая очередной
объект String, мы не прописываем полное имя класса - java.lang.String.
    Для доступа к классам подпакета(пакет в пакете) нужно прописывать полное имя(смотри Subpackage).

    Для чего нужен import static
    Тоже самое, что и импорт, только статических элементов.

    При любом вызове переменной, или метода, или еще-чего нибудь JVM прежде всего ищет в самом классе, если не находит, то обращается
к импортам.
